Requirement 1 game plan

1. Collect input
   - Ask the user for the business scenario.

2. Define a canonical output contract (before prompting)
   - Create Pydantic models for Schema, Table, Column, PrimaryKey, and ForeignKey.
   - Enforce strict enums and field definitions for required types (numerical, categorical, semi-structured such as JSON/XML/text).
   - Require the model to return JSON only in this contract.

3. Assemble prompt and generate schema
   - Build system prompt using Requirement 1 requirements plus the user scenario.
   - Prompt Gemini (or configured LLM) to generate schema JSON that matches the contract.

4. Validate generated schema
   - Parse output with Pydantic.
   - Run custom checks for:
     - multiple tables,
     - primary keys,
     - foreign keys with valid table/column references,
     - required field-type coverage,
     - duplicate table/column names.

5. Deterministic post-processing
   - Normalize names to a consistent convention.
   - Normalize type aliases to canonical types.
   - Sort entities for stable output.

6. Retry loop with guardrails
   - If validation fails, return structured errors to the LLM and regenerate.
   - Set a retry cap (for example, max_attempts = 3) to avoid infinite loops.
   - If retries are exhausted, return the latest schema candidate plus a clear failure report.

7. Output artifacts
   - Output canonical schema JSON.
   - Render SQL DDL from canonical schema.
   - Include a validation summary report.

8. Suggested module layout
   - scenario_parser.py
   - schema_generator.py
   - schema_validator.py
   - schema_normalizer.py
   - schema_renderer.py
